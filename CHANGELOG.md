## 2026-07-15 — 코드리뷰 6차: 글리프 커버리지 · 파비콘 · 배포 위생

### ⚠️ 글리프 누락 3건 — 리뷰는 "서브셋 실패"로 묶었으나 원인이 셋 다 다름
| 글자 | 원본 폰트에 | 진짜 원인 | 조치 |
|---|---|---|---|
| `&` | **있음** | **내 추출 버그** | ✅ 수정 |
| `✻` U+273B | **Pretendard에 없음** | 폰트 자체 한계 | ✅ `※`(U+203B)로 교체 |
| `·` (Gaegu) | **Gaegu에 없음** | 폰트 자체 한계 | ❌ 불가 — 폴백 유지 |

- **서브셋 탓은 `&` 하나뿐.** `·`·`✻`는 원본에 없어 **CDN 시절에도 똑같이 폴백**이었음 = 새 문제 아님. 서브셋으론 없는 글리프를 만들 수 없다.
- **`&` 원인 = `re.sub(r'&[a-z]+;',' ',t)` 로 엔티티를 공백 처리** → 소스의 `&amp;` 가 `&` 로 안 풀려 문자셋에서 누락. **`html.unescape()` 를 썼어야 함.**
  - ⚠️ **[[palette-warm-beige-drift]] 의 파비콘 실패와 같은 계열.** 거기선 hex 가 `%23` **URL 인코딩**이라 grep 을 빠져나갔고, 여기선 `&` 가 `&amp;` **HTML 엔티티 인코딩**이라 빠져나갔다. **소스형 ≠ 렌더형** — 형태만 바꿔 같은 실수를 반복했음.
- **`✻` 두부(□) 위험은 실재**했음: Pretendard·Gaegu 둘 다 없어 시스템 폴백인데, 딩뱃이라 없는 기기면 □. `※`는 Pretendard 에 있어 안전.
- **커버리지 전수 검증 추가**: 렌더 텍스트 전 문자 vs 서브셋 cmap 대조 → **Pretendard 400/600 누락 0**, Gaegu 는 `·` 1개(원본 한계, 폴백 정상).
